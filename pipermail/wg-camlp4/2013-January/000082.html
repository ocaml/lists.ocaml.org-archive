> For example (a quotation for Haskell's arrow notation):
>
> <:proc < x ->           y <- f -< x + 1;
>           g -< 2 * y;
>           t <- h -< x * (x + y);
>           return -< t + y >>
>
> is definitely preferable to:
>
> <:proc < $x$ ->           y <- $f$ -< $x + 1$;
>           $g$ -< $2 * y$;
>           t <- $h$ -< $x * (x + y)$;
>           $return$ -< $t + y$ >>
>

FWIW: You can generate the second from former, by making a pass over the AST.

-Wojciech
