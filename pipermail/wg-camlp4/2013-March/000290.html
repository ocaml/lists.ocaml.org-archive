I've committed a version where attributes are mostly parsed as in your 
patch.

Currently supported:

- Attributes  .... [: lid expr ]
   where .... can be:

     a module expression
     an expression
     a type expression

- Extension (: lid expr )
   in position of a:

    a module expression
    an expression
    a type expression

- Attributes .... with lid1(expr1),..,lidn(exprn)
   where .... can be:

    a single type declaration
    an open or include structure item (more to come)
