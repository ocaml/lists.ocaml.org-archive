 > Hi.<br>
 ><br>
 > I also had need of such a library when I started working on a<br>
 > distributed programming project, but as I couldn't find anything that<br>
 > fulfilled my requirements, I wrote my own.<br>
 ><br>
 > The library is Lwt-based, and provides pretty straightforward remotely<br>
 > callable procedures:<br>
 ><br>
 > type ('a, 'r) handle<br>
 > val ($) : ('a, 'r) handle -> 'a -> 'r Lwt.t<br>
 > val publish : ('a -> 'r Lwt.t) -> ('a, 'r) handle<br>
 ><br>
 > The library is designed to work in a firewalled environment: firewalled<br>
 > nodes can join the distributed network by connecting to any accessible<br>
 > host of that network with an open listener. All messages between nodes<br>
 > are routed through the network.<br>
 ><br>
 > type connection<br>
 > val connect :<br>
 > ?host:string -> ?port:int -> unit -> connection Lwt.t<br>
 > val listen :<br>
 > ?port:int -> unit -> unit Lwt.t<br>
 ><br>
 > Initial values are obtained with "roots". A root (usually a handle, or a<br>
 > record or first-class module containing handles) is keyed to a string,<br>
 > and once set by a node, it can be discovered by any node in the network:<br>
 ><br>
 > val get_root : string -> 'a Lwt.t<br>
 > val set_root : string -> 'a -> unit Lwt.t<br>
 ><br>
 > This is the only unsafe part of the interface: the getter and setter<br>
 > _must_ agree on the type of the root or all hell breaks loose.<br>
 ><br>
 > I haven't published the library yet, since I have wanted to feel free to<br>
 > modify it as required by my application, but if you are interested, and<br>
 > cannot find a more mature solution, I'd be happy to provide it for your<br>
 > inspection.<br>
