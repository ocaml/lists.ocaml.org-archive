    There is no setup.ml files in the current Bitbucket repo of Core,
    only an `oasis.sh' script for each specific library, so you need to
    generate all the configure files yourself.<br>
    <br>
    I don't know whether Sylvain subscribed to this list, but I got help
    from him via email.  The whole story is:<br>
    <br>
    There is actually a patch for the 0.2.1~alpha1 release to handle the
    pack option, contributed by Adrien Nader, who told me about this on
    #ocaml IRC channel.<br>
    <br>
<a class="moz-txt-link-freetext" href="http://www.forge.ocamlcore.org/tracker/download.php/54/294/573/151/0001-Apply-for-pack-patch-and-make-tests-work.patch">http://www.forge.ocamlcore.org/tracker/download.php/54/294/573/151/0001-Apply-for-pack-patch-and-make-tests-work.patch</a><br>

            <div text="#000000" bgcolor="#ffffff"> For those interested,

              Adrien Nader and Sylvain Le Gall)<br>

              2) uninstall previously installed version of
              type-conv/sexplib/bin_prot/fieldslib , etc.<br>
              <br>
              3)  uncomment all `Pack:true' in every oasis.sh script in
              the directory tree , if any<br>
              <br>
              4)  patch base/core/lib/backtrace_stubs.c to compile on
              32-bit machines<br>
