I've observed that names for e.g. source directories, Oasis packages,<br>
libraries, etc., are not always consistent in the Core distribution.<br>
I think it shouldn't be the case as it is right now that an Oasis<br>
package whose source is in directory "bin-prot" is named "bin_prot",<br>
it's documentation "bin-prot", and the library "bin_prot" again.<br>
<br>
If we wanted to make all names consistent, we would have to convert<br>
dashes to underscores, because dashes are illegal in module names (and<br>
hence names for packed libraries).  Underscores would work in any<br>
context.<br>
<br>
We should probably settle on naming conventions before creating a<br>
public release.  Such changes would be fairly straightforward now, but<br>
likely inconvenient later.<br>
<br>
What do you think, should all these names be made consistent?  If so,<br>
future projects should also make sure to use a consistent naming<br>
convention, in particular to always use underscores instead of dashes<br>
in all names.<br>
