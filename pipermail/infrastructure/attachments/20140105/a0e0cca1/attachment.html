<div dir="ltr"><div>I hope this does not sound too negative, but I'm not sure we would gain by "blessing" tuareg as "the" ocaml editor mode for Emacs. The parallel existence of caml-mode and tuareg-mode has been a long-time annoyance of mine, because I don't see a strong justification for using either instead of the other. In particular tuareg has had long-standing bugs that are rather annoying, and I for one would be very happy to throw away any kind of elisp-implemented approximation of the OCaml syntax, and use a more principled approach such as Merlin+ocp-index to handle them (none of which currently support syntax highlighting iirc., but that could be  added to ocp-index capabilities).<br>
