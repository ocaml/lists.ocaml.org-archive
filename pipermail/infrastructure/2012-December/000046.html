> Hi,
> 
> in the past days I had a lot of fun mass-building OASIS packages. View the results here:
> 
> https://godirepo.camlcity.org/openapps/autoui.cgi?page=overview
> 
> Scroll down until the package names begin with "oasis-".
> 
> This is not so bad:
> 
> - 69 good packages (green or "duplicate", i.e. a package with same
>   contents is already part of GODI and cannot be installed, but
>   build is ok)
> - 13 packages not configurable
>   * 8 of these have problems with config switches, typically
>     --destdir, but also others
>   * 3 of these cannot create a working setup.ml
>   * 2 do not find prerequisites (this can be an import problem)
> - 19 package not buildable
>   * various problems, often the version seem to be outdated and
>     unsuited for ocaml-4.00
> - 1 package not installable
>   * this one tried to install stuff outside the allowed directories
> - 2 packages not downloadable
>   * this seems to be a problem with oasis-db API. Sometimes tarballs
>     contain escape characters (%7E instead of ~)
> - 7 packages with hard packaging problems
>   * e.g. camlzip has a negative "ord" number
>   * these are the blank fields in the table. Data is available in
>     log files, though
> - 13 packages have errors in prerequisite packages ("skipped")
> 
> total: 125 packages have db entries here
>
