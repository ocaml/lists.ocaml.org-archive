"><br class="">### Who is still using opam 1.2.0?<br class=""><br class="">I found that the Debian Jessie (stable) release includes 1.2.0, and this is probably the last major distribution including it.  The [Debian Stretch](<a href="https://wiki.debian.org/DebianStretch" class="">https://wiki.debian.org/DebianStretch</a>) is due to become the stable release on the 17th June 2017, and so at that point there will hopefully be no distributions actively sending opam 1.2.0 out.<br class=""><br class="">Is there anyone else that is still packaging 1.2.0?  Please comment here if so, and we should move them away.<br class=""><br class="">### How do we deprecate it?<br class=""><br class="">Due to the format changes happening in a minor version, it's a bit difficult to give opam 1.2.0 users a smooth transition experience, to my knowledge (Louis Gesbert or Thomas Gazagnaire may correct me here).  I would propose:<br class=""><br class="">- putting a notice on <a href="http://opam.ocaml.org" class="">opam.ocaml.org</a> that 1.2.2 is the only supported stable release.<br class="">- can we somehow put in a request to debian-stable to add a post-installation message that the upstream package will no longer work since the repository? <br class=""></div></div></blockquote></div><br class=""><div class="">Thank you all for the helpful feedback. Since there have been no more messages or objections, I have started the process off here:</div><div class=""><br class=""></div><div class=""><a href="https://github.com/ocaml/opam-repository/pull/9477" class="">https://github.com/ocaml/opam-repository/pull/9477</a></div><div class=""><br class=""></div><div class="">When I merge it, I will also update <a href="http://opam.ocaml.org" class="">opam.ocaml.org</a> with a deprecation notice.  If you do spot any distributions packaging opam 1.2.0, please do feel free to contact them to request an upgrade to a supported release such as 1.2.2.</div><div class=""><br class=""></div><div class="">reg
